# 📱 PHOBBY - 스마트폰 리뷰 기반 맞춤형 추천 시스템

스마트폰 구매를 고민하는 사용자에게 YouTube/다나와 리뷰 데이터를 분석하여 취향에 맞는 기종을 추천하는 웹 서비스입니다.

## 프로젝트 개요
- 진행 기간 : 2025.01.23 ~ 2025.03.28
- 목표 : 리뷰 데이터를 활용해 사용자의 취향을 반영한 스마트폰 자동 추천 시스템 구축

| 이름       | 역할              | 담당 업무                                                                                    |
| -------- | --------------- | ---------------------------------------------------------------------------------------- |
| 신지수 (본인) | 데이터 엔지니어링 / 백엔드 | ETL 파이프라인 구축, Airflow 자동화, 데이터 수집과 전처리, Django 백엔드 개발, RAG 기반 추천 시스템 설계, 전체 아키텍처와 ERD 설계 |

## 문제 해결 및 개선 요약

### 1) 데이터 품질 문제

**문제**
- 리뷰 데이터에 은어, 비속어 등 노이즈가 많음
- 스마트폰 도메인 용어를 고려하지 못해 핵심 문장이 삭제됨
- 전처리 오류로 유효 데이터가 손실되고 분석 정확도 저하

**해결**
- 스마트폰 관련 핵심 용어들이 포함된 문장은 제거되지 않도록 필터링 규칙을 설계하여 데이터 품질 높임 

### 2) 수집 비용 및 제한 문제

**문제**
- YouTube API 제한과 Selenium 속도로 재수집 부담 증가
- 원본 미보존으로 실패 시 매번 처음부터 재수집해야 하는 비효율 발생
  
**해결**
- 모든 수집 데이터를 HDFS Raw 형태로 저장하도록 구조 개선
- 한 번 수집한 데이터를 기반으로 전처리 및 분석을 여러 번 수행할 수 있도록 함
- 원본이 보존되기 때문에, **수집 과정에서 실패가 발생해도 재수집 없이 즉시 재처리 가능하도록 개선**

### 3) 성능 문제

**문제**
- 웹 서비스 개발 과정에서, 사용자가 분석 요청을 보낼 때마다 Django 서버가 평균 6분 이상(374초) 응답이 지연되는 성능 문제가 발생
- 로깅을 통해 병목 구간을 분석한 결과, 데이터 로딩 및 전처리(121초)와 브랜드 분류(231초)가 대부분의 시간을 차지함
- 원래는 전처리와 브랜드 분류 같은 무거운 가공 작업이 Airflow에서 사전에 처리되어 DB에 적재되어야 했지만, 기간 제약으로 Django가 실시간으로 이를 수행하면서 성능 문제가 발생함

**해결**
- 전처리 결과와 브랜드 분류 결과를 MySQL 가공 테이블로 분리해 중복 연산 제거
- 전체 처리 시간 **374초 → 34초로 약 90% 감소**
- LLM 요약과 연관어 분석을 Celery 비동기 처리로 분리
- 분석 기능을 3개 페이지로 나눠 로딩 부담 감소
- Django는 즉시 응답하고 Celery가 백그라운드에서 분석 후 결과를 DB에 저장하는 구조로 개선

**성과**
- 전체 처리 시간 **374초 → 34초(약 90% 개선)**
- Django 초기 응답 속도 **6분 → 1초 미만으로 단축**
- 페이지 단위 로딩으로 사용자 접근성 향상
